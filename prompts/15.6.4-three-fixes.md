## Issue 1 — Empty gray placeholder in PushPilot section

**Symptom:** After the "PushPilot · Gates 6" section divider, a large empty gray rectangle renders above the per-BU PushPilot cards. Looks like a loading skeleton that never fills.

**Root cause hypothesis:** The PushPilot section was coded assuming an "aggregate PushPilot decision" event arrives first, then per-BU decisions stream in. In reality, PushPilot only emits per-BU decisions (one per BU that passed Gate 5). The aggregate placeholder is reserved but never populated.

**Fix:** Remove the aggregate placeholder. The section should render its divider/header on mount, then populate with per-BU cards as `pushpilot_decision` events arrive (one per BU). Set shimmer count to 0 for pushpilot in `renderAgentStarted`.

## Issue 2 — Duplicate terminal state heading (critical regression)

**Fix (this time actually land it):**

For ALL terminal states (not just AWAITING_HITL): pass empty string as section title. Update `createSection` to conditionally omit the section-title div when title is empty. The body heading (`terminal-summary__title`) carries the state name as the sole visual occurrence.

For AWAITING_HITL: keep trigger type in section bar (already from 15.6.3), keep no body heading.

**Verify visually** — run all five scenarios. State name appears exactly once.

## Issue 3 — PushPilot confidence shows 0.00

**Symptom:** Every PushPilot decision card shows a confidence value of `0.00` with an empty bar track.

**Investigation:**

The `pushpilot_decision` event's `preference` dict has `verb`, `channel`, `reason` — but NO `confidence` key. The frontend reads `p.preference.confidence` → undefined → renders as 0.00.

**Three likely paths:**

- **Path A:** PushPilot doesn't produce a confidence score — hide bar entirely.
- **Path B:** Schema has confidence but event handler isn't extracting it.
- **Path C:** Confidence is in a different structural location.

**My lean: Path A.** PushPilot's Gate 6 is a timing/channel decision driven largely by deterministic policy. Hiding the bar is architecturally honest.

**Apply Path A fix:** Omit `.confidence-bar` and `.confidence-value` elements from the non-diverged PushPilot card. Keep verb badge + gate label + reasoning.
