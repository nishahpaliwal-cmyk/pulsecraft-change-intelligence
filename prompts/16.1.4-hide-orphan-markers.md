# Prompt 16.1.4 — Hide Orphan Arrow Markers

> **Character.** SVG rendering gotcha. Arrow marker heads (the triangle tips) render immediately even when the arrow line itself is hidden behind `stroke-dashoffset` animation. Result: floating triangle markers appear in empty space before their nodes have arrived. Hide markers until their arrow is drawing.
>
> **How to use.** Paste below the `---` line into Claude Code. Fully autonomous.
>
> **Expected duration:** 10-15 minutes.
>
> **Budget:** Zero LLM cost.

---

# Instructions for Claude Code

Visible bug on the Architecture tab: during the entrance animation, arrow marker triangles appear in empty space before their corresponding arrows draw, and before the nodes at either end of those arrows have appeared.

Root cause: SVG `<marker>` elements attached to `<line>` or `<path>` elements render independent of the line's `stroke-dashoffset` animation. Hiding a line via `stroke-dashoffset` does not hide its attached marker.

Fix: make markers opacity 0 by default, fade them in during (or at the end of) their parent arrow's draw animation.

## Approach used (B — per-edge dataset)

Store each edge's `markerUrl` in `dataset.markerUrl` at draw time (`_drawEdges()`). At the start of the full animation, set all edges' `marker-end` to `"none"`. In `drawEdge()`, schedule a `setTimeout` at `+dur` ms to restore the correct `marker-end` once the `stroke-dashoffset` transition has completed.

Replay resets via the same `"none"` sweep before re-running the sequence.

`prefers-reduced-motion` path: reduced-motion branch restores `marker-end` from dataset alongside `stroke-dashoffset` (markers visible immediately in final state, no hide/reveal cycle).

## Files changed

- `src/pulsecraft/demo/static/architecture.js` — three targeted edits:
  1. `_drawEdges()`: add `path.dataset.markerUrl = markerUrl` after `setAttribute('marker-end', markerUrl)`
  2. Reduced-motion branch of `_runEntrance()`: restore `e.setAttribute('marker-end', mu)` alongside dashoffset reset
  3. Full animation section: add `_svg.querySelectorAll('.arch-edge').forEach(e => e.setAttribute('marker-end', 'none'))` before the timed sequence; update `drawEdge()` to schedule marker restoration at `+dur` ms
