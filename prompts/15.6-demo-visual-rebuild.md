### Motion principles — "dynamic and responsive"

Every meaningful state change has motion. Motion carries meaning:

- **Arrival motion** — new content enters with purpose. Not sudden (feels abrupt) and not slow (feels sluggish). 300-500ms with an ease-out curve.
- **Departure motion** — old content leaves cleanly. 200-300ms.
- **Interaction motion** — hover, click, focus produce immediate responsive feedback. 150-200ms.
- **Progress motion** — pipeline progress is continuously visible through subtle persistent motion (progress rail dots pulse while active, confidence bars fill with weight).
- **Stagger** — when multiple similar elements arrive together, they stagger by 60-100ms. Never all at once.

### Animation timing reference

| Element | Property | Timing | Easing |
|---|---|---|---|
| Section divider appears | opacity + translateY(-8px → 0) | 350ms | cubic-bezier(0.2, 0.8, 0.2, 1) |
| Decision card appears | opacity + translateY(12px → 0) | 420ms | cubic-bezier(0.2, 0.8, 0.2, 1) |
| BU card appears (sync across BUs) | opacity + scale(0.96 → 1) | 480ms | cubic-bezier(0.2, 0.8, 0.2, 1) |
| Confidence bar fills | width (0 → value) | 700ms | cubic-bezier(0.34, 1.2, 0.64, 1) *slight overshoot* |
| Verb badge appears | opacity + scale(0.9 → 1) | 250ms | cubic-bezier(0.2, 0.8, 0.2, 1) |
| Hook badge pulse | box-shadow + scale | 800ms total | ease-in-out |
| Progress rail dot activates | background-color + scale(1 → 1.2 → 1) | 600ms | ease-out |
| Progress rail dot completes | background-color | 250ms | ease-out |
| Scenario card hover lift | translateY + box-shadow | 180ms | ease-out |
| Scenario card active state | border-color + background | 220ms | ease-out |
| Audit drawer slide in | translateX(100% → 0) + opacity | 360ms | cubic-bezier(0.2, 0.8, 0.2, 1) |
| Welcome state exit | opacity + scale(1 → 0.98) | 280ms | ease-out |
| Cost/elapsed counter tick | opacity of new digit | 180ms | linear |
| Message preview compose | opacity + translateY per element | 300ms each, 80ms stagger | ease-out |

**Staggering:** within a section, siblings arrive with a **60ms stagger** between them. The stagger is a big part of what makes multiple arrivals feel choreographed rather than chaotic.
